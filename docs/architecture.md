# Architecture

High-level map of the repository. Older narrative notes (RTL/Cairo wording, auth details) live in [**ARCHITECTURE-legacy.md**](./ARCHITECTURE-legacy.md); some sections there are **stale** (e.g. fonts now IBM Plex in `app/layout.tsx`, dashboard uses `getServerSession`). Prefer this file + [rtl-conventions.md](./rtl-conventions.md) for current behavior.

## Top-level layout

| Path | Role |
|------|------|
| **`/app`** | Next.js App Router: layouts, pages, route handlers, global styles. |
| **`/components`** | React components: shared UI (`components/ui`), layout shell, feature modules under `components/modules/*`. |
| **`/actions`** | Server Actions (`"use server"`): mutations and data loaders for dashboard features. |
| **`/lib`** | Shared libraries: DB client, schema re-export, auth, i18n helpers, utilities. |
| **`/lib/db`** | Drizzle schema (`schema.ts`) — not a separate top-level `/db` folder. |
| **`/drizzle`** | SQL migrations generated by drizzle-kit (`drizzle.config.ts` → `out: "./drizzle"`). |
| **`/messages`** | next-intl JSON catalogs (`ar.json`, `en.json`). |
| **`/hooks`** | Client hooks (`use-mobile`, `use-translate-action-error`, etc.). |
| **`/types`** | Shared TypeScript types. |
| **`/scripts`** | One-off scripts (`seed-admin.ts`, `fix-migrations.ts`). |
| **`/public`** | Static assets (logos, manifest). |
| **`/app/api`** | API routes: auth, upload, search, PDF, scrape helpers. |

## `/app` — App Router

- **`app/layout.tsx`** — Root layout: fonts (IBM Plex Arabic / Latin), `dir` + `lang` from next-intl locale, `NextIntlClientProvider`, `Providers`.
- **`app/globals.css`** — Tailwind v4 theme tokens; sidebar forced to dark/black palette.
- **`app/(guest)/`** — Login, 404/500 pages.
- **`app/dashboard/`** — Authenticated shell: `layout.tsx` (`getServerSession`, `SidebarProvider`, `AppSidebar`, `SiteHeader`, `MobileBottomNav`).
- **`app/dashboard/*/page.tsx`** — Feature pages: clients, projects, tasks, invoices, expenses, proposals, team, reports, services, settings, my-tasks.
- **`app/api/*`** — REST-style handlers (upload, search, invoice PDF, locale cookie).

## `/components`

- **`components/ui/`** — shadcn primitives (button, dialog, table, **sidebar**, **command**, **date-picker-ar**, sortable table, etc.).
- **`components/layout/`** — `header`, `sidebar`, `search` (⌘K palette for static routes), `logo`.
- **`components/modules/*`** — Feature UIs: clients, projects, tasks, invoices, expenses, proposals, team, services, files, reports.
- **Root-level** — `app-sidebar`, `site-header`, `global-search`, `mobile-bottom-nav`, dashboard widgets, charts.

## `/actions` — Server Actions

Server Actions live in the root **`/actions`** directory (20 `.ts` files). See [server-actions.md](./server-actions.md).

Files include: `assignments.ts`, `clients.ts`, `dashboard.ts`, `expenses.ts`, `files.ts`, `invoices.ts`, `project-services.ts`, `projects.ts`, `proposals.ts`, `reports.ts`, `services.ts`, `settings.ts`, `tasks.ts`, `team.ts`, `team-members.ts`.

## `/lib`

| File / dir | Role |
|------------|------|
| **`lib/db/index.ts`** | Neon HTTP Drizzle client + re-exports schema. |
| **`lib/db/schema.ts`** | Drizzle PostgreSQL schema, enums, relations. |
| **`lib/db-errors.ts`** | `isDbConnectionError`, `getDbErrorKey` for Neon/network failures. |
| **`lib/i18n-errors.ts`** | `isDbErrorKey` — narrows string to `DbErrorKey` for translators. |
| **`lib/auth.ts`** | NextAuth options. |
| **`lib/locale.ts`** | Server actions: read/set `locale` cookie (`ar` / `en`). |
| **`lib/imagekit.ts`** | ImageKit upload helpers. |
| **`lib/routes-config.tsx`** | Nav items for sidebar / command menu. |
| **`lib/currency.ts`**, **`lib/settings-schema.ts`**, **`lib/table-views.ts`**, **`lib/table-module-map.ts`** | Domain helpers. |

## `/drizzle` — migrations

- Generated SQL migrations (`0000_*.sql` … `0003_*.sql`).
- **`drizzle.config.ts`** points `schema` to `./lib/db/schema.ts` and `out` to `./drizzle`.
